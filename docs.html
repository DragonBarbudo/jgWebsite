Instalación

Descargar Dragón Webeloper de los links superiores.

Una vez descargada cambiar el nombre "DragonWebeloper2" por el nombre del proyecto.

La carpeta deberá ser añadida a un compilador de JADE y LESS (Prepros) y abierta en un editor de código (Brackets, Atom).

Si el compilador solicita configuración se deberá desactivar el análisis de JS y activar el archivo site/css/style.less como LESS principal para compilar.

Pasar a Configuración

Configuración

La carpeta dw contiene el núcleo de DragonWebeloper y no debe ser alterado a no ser que se necesite alguna configuración no habilitada desde esta documentación.

La carpeta: site se encarga de mantener los recursos (multimedia) y archivos como CSS / JS, así como inclusión de librerías y plugins necesarios en específico para el proyecto.

Organización en "site".

Dentro de la carpeta site los elementos se deberán organizar de la siguiente manera:

  • css | Contiene el css principal "style.less" en el cual se trabajará. Este archivo es compilado originando "style.css".
  • js | Se trabaja sobre "script.js" que al mismo tiempo incluye acciones de plugins pre-instalados para ejecutar acciones comunes de Dragon Webeloper.
  • img | Aquí se añadirá el total de imagenes a ser utilizada dentro del proyecto.

Configurar proyecto

En la carpeta site se encuentra el archivo "_config.jade". Dentro se pueden definir datos rápidos de uso general así como el menu principal del sitio.

- var domain

Define el ROOT del sitio. En local deberá funcionar como "index.html". Si es cargado a un servidor deberá cambiarse por el dominio de éste: "http://dominio.com".

- var title

Título del sitio

- var keywords

Palabras clave del proyecto / sitio

- var description

Descripción del sitio

STYLESHEETS

Siguiente la misma estructura se pueden añadir hojas de estilo que se encuentren dentro de la carpeta site (site/css/hoja.css) o externas en un servidor http.

JAVASCRIPTS

Igual que Stylesheets

MENU

Siguiendo la misma estructura se pueden añadir o quitar menus generales del proyecto.

La sintáxis de uso es:

- menu.push(['NombreDeLaPágina', 'NombreDelArchivo'])

El nombre de la página se refiere al texto que será mostrado en el sitio.

El nombre del archivo (slug) es el nombre del archivo que será compilado al html (archivo.jade > archivo.html) y contiene la estructura.

Jade

Dragon Webeloper 2 esta basado casi en su totalidad en el lenguaje de Jadeque es una sintáxis para compilar HTML's y que mezcla características de javascript.

En su versión nativa corre sobre Node.js, sin embargo, puede ser utilizado mediante un "Precompilador" como los antes mencionados (Prepros, Codekit...).

Para conocer más sobre el lenguaje ingresar al sitiohttp://jade-lang.com/

Responsive grid

Las columnas inicialmente trabajan con el tamaño máximo a 960px pero cambian a 480px cuando la resolución de la ventana es menor a 960px. Cuando la resolución es menor a 480px el .container cambia su ancho al 100%, por lo que se vuelve un diseño fluído.

Estos tamaños se pueden manipular modificando las variables @containerWidth y @containerWidthTablet de site/css/style.less.

Mediante los elementos -t y -m se pueden establecer diferentes acciones en Col y Push.

Dispositivo Resolución Prefijo .container
Desktop Mayor a 960px .col1 y .push1 960px
Tablet Entre 480px y 960px .col-t1 y .push-t1 480px
Móvil Menor a 480px .col-m1 y .push-m1 100%

Navegación

El menú principal es generado desde el archivo "_config.jade". Ver Configuración para más detalles.

El mixin que genera la creación de un menú de navegación es:

+menu()

El método de uso es el siguiente:

+menu('PaginaActiva', menu, 'TipoDeMenu', 'Prefijo', 'Sufijo', 'ClaseParaLinks', {Atributos})

'PaginaActiva'

Indica el nombre de la página en la que se encuentr actualmente. Agrega la class "active". Se puede utilizar para resaltar el botón.

menu

Es la variable definida en el archivo "_config.jade". Puede crearse una variable nueva (de tipo array) y crear un menú diferente, teniéndo múltiples menus.

'TipoDeMenu'

Actualmente existen "horizontal" y "vertical". El menú horizontal distribuye el número de elementos (links) en "cols" del Grid. El menú "vertical" elimina los bullets.

+menu('index', menu, 'horizontal')
  • Inicio
+menu('index', menu, 'vertical')
  • Inicio

'Prefijo' y 'Sufijo'

Prefijo Añade texto o símbolos antes del "slug" dentro del link. Útil para añadir # al principio del link. (Usado en navegación por scroll).

Sufijo agrega ".html" al final del "slug" si nada es definido. Para colocar texto por 'default' (.html) se puede dejar sin contenido o colocar un 0. Si se quiere un sufijo nulo escribir comillas con espacio " ".

+menu('index', menu, 'horizontal', '#', ' ')
  • Inicio


El link generado para los botón es : #slug

'ClaseParaLinks'

Añade un class a cada elemento <a class=""> del menú.

+menu('index', menu, 'horizontal', '#', ' ', 'btn small')
  • Inicio

Cada link tiene .btn.small

{Atributos}

Permite añadir atributos al elemento <a data=""> del menú utilizando una sintáxis de JSON.

Por ejemplo: {'data':'valor'} | {'data-scroll':true} | {'data-sr':'enter right'}

+menu('index', menu, 'horizontal', 0, 0, '', {'data-sr':'enter right'})
  • Inicio


Cada link tiene animación mediante el attributo data-sr. Ver módulo "Animaciónes".

Formularios

Los formularios trabajan con el API de Mandrillen conjunto con un sistema en PHP cargado en https://dragonbarbudo.com/api/email.php.

El action="" del form se redirige hacia la api de dragonbarbudo.com (link anterior) y se mandan inputs de tipo hidden con información de receptor(_to), emisor(_from) y asunto(_subject):

form(action="https://dragonbarbudo.com/api/email.php", method="get")
   input(type="hidden", name="_to", value="receptor@dominio.com")
   input(type="hidden", name="_from", value="emisor@dominio.com")
   input(type="hidden", name="_subject", value="Asunto del correo")

Si se utiliza envío mediante ajax no es necesario especificar nada más. Sin embargo este desarrollo debería ser manual.

Si se desea utilizar el método tradicional se deberá especificar la ruta a la que se redireccionará después en un campo _redirect

   input(type="hidden", name="_redirect", value="http://dominio.com/mensajeEnviado.html")

MIXIN para crear campos rápidos:

+formInput('Tipo', 'Name', 'Label:', 'Placeholder', Requerido)

'Tipo' text, textarea, date, number, tel, email...

'Name' El correo recibido hace referencia a este campo para señalar el contenido.

'Label' Etiqueta que se muestra antes del input.

'Placeholder' Texto dentro del campo que desaparece al escribir.

Requerido| false o true. Indica si el campo es necesario.

+formInput('text', 'Empresa', 'Empresa:', '¿Dónde trabajas?', false)

Método express

Simple

Se puede crear un formulario rápido (simple) con los campos tradicionales: nombre, correo, teléfono y mensaje:

+formSimple('receptor@domino.com', 'emisor@dominio.com', 'Asunto del mensaje')
 

Simple extendido

Es posible extender el formulario con campos personalizados. Los nuevos campos aparecerán antes de la caja de mensaje y pueden utilizarse estructura del GRID:

+formSimple('receptor@domino.com', 'emisor@dominio.com', 'Asunto del mensaje')
   .container
     .col6
       +formInput('date', 'Nacimiento', 'Fecha de nacimiento:', '', true)
     .col6
       +formInput('text', 'Empresa', 'Empresa:', '¿Dónde trabajas?', false)
 

Imagen de fondo | Backstretch

La configuración del plugin se encuentra en site/js/script.js

Este módulo permite asignar 1 o múltiples imagenes (en slide) de fondo a cualquier elemento o al body mismo. Mantiene el tamaño de la imagen de modo que se ajuste al tamaño del contenedor.

+sliderbg('Target', 'Imagen')

'Target'

Es el elemento al que se le añadirá la imagen de fondo. Puede ser 'body' para agregarlo al fondo del sitio o un elemento específico ya sea por su ID ('#elemento') o CLASS ('.elementos')

'Imagen'

Al igual que el módulo de +slider, este componente permite añadir imagenes al infinito.

1 Imagen de fondo:

+sliderbg('#target', 'rutadeimagen.jpg')
#target1

Slider de fondo:

Al añadir más imagenes trabaja como un slider de fondo que va realizando el paso por cada imagen.

+sliderbg('#target', 'rutadeimagen.jpg', 'rutadeimagen2.jpg')
#target2

Se puede utilizar con cualquier contenedor.

Animaciones | ScrollReveal

La configuración del plugin se encuentra en site/js/script.js

Añade animacion de entrada a cualquier elemento al añadir la propiedad "data-sr". La animación ocurre cada vez que el objeto sale y entra de la ventana.

div(data-sr) Animación
Animación

Se puede configurar de diferentes maneras:

(data-sr)
(data-sr="enter top")
(data-sr="move 50px enter left")
(data-sr="enter right move 50px")
(data-sr="wait 0.8s flip 90deg")
(data-sr="scale up 100% wait 0.5s")
(data-sr="roll -45deg over 3.0s")
(data-sr="spin 180deg")
